The admin dashboard is the control centre for the SENAshopping platform. It displays real-time statistics about users, entrepreneurs, products, and favorites, and provides quick access to user and product management.

<Info>
  The dashboard is protected by `ProtectedRoute` with `requiredRole="administrador"`. Any user without the `administrador` role is redirected to `/` when they try to access `/admin`.
</Info>

## Accessing the dashboard

<Steps>
  <Step title="Log in as an administrator">
    Navigate to `/auth` and sign in with an account that has the role `administrador`. The JWT token and user object are stored in `localStorage` under the keys `token` and `user`.
  </Step>
  <Step title="Open the dashboard">
    Navigate to `/admin`. The `ProtectedRoute` component reads `localStorage` and verifies both that a token exists and that `user.role === "administrador"` before rendering the page.

    ```jsx
    // src/components/ProtectedRoute/ProtectedRoute.jsx
    const ProtectedRoute = ({ children, requiredRole }) => {
      const token = localStorage.getItem("token");
      const user = JSON.parse(localStorage.getItem("user"));

      if (!token) return <Navigate to="/auth" replace />;

      if (requiredRole && user?.role !== requiredRole) {
        return <Navigate to="/" replace />;
      }

      return children;
    };
    ```
  </Step>
</Steps>

## Statistics API

On mount, the dashboard fetches all statistics from a single endpoint:

```javascript
// src/pages/Admin/Dashboard.jsx
useEffect(() => {
  fetch("http://localhost:3000/api/admin/stats", {
    headers: { Authorization: `Bearer ${token}` },
  })
    .then((res) => res.json())
    .then(setStats)
    .catch(() => setStats(null))
    .finally(() => setLoading(false));
}, []);
```

| Field | Type | Description |
|---|---|---|
| `totalUsuarios` | number | Total active registered users |
| `totalEmprendedores` | number | Total active entrepreneur accounts |
| `pendientes` | number | Accounts awaiting approval |
| `totalProductos` | number | Total published products |
| `totalFavoritos` | number | Total favorites saved across all users |
| `productosPorCategoria` | array | Each category with a `categoria` and `total` field |
| `ultimosUsuarios` | array | Most recently registered users |

## Products by category

Below the summary cards, a bar chart section shows how products are distributed across categories. Each row displays:

- The category name
- A proportional fill bar calculated as `(cat.total / stats.totalProductos) * 100%`
- The raw product count for that category

This gives an instant visual sense of which categories dominate the catalogue without leaving the dashboard.
