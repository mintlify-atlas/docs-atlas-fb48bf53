Sena Shopping is a marketplace frontend built for SENA (Servicio Nacional de Aprendizaje) entrepreneurs to publish and sell their products to customers across Colombia. The application is a single-page app written in React 19, bundled by Vite 7, and communicates with a Node.js REST API over HTTP. Entrepreneurs register their products, customers browse and save favorites, and administrators moderate the platform from a dedicated dashboard.

<Info>
  Sena Shopping is a frontend-only application. It requires a separate backend REST API running at `http://localhost:3000/api` to handle authentication, product data, and user management.
</Info>

## User roles

The platform defines three distinct roles. A user's role is set at registration time and is stored inside the JWT payload and the `user` object in `localStorage`.

Protected routes use the `<ProtectedRoute>` component, which reads the stored JWT and optionally checks the `requiredRole` prop before rendering the page. Attempting to access `/admin` without the `administrador` role redirects the user away.

## Backend API connection

All API calls target `http://localhost:3000/api`. The auth service (`src/services/authService.js`) targets the `/auth` namespace directly:

```javascript src/services/authService.js
const API_URL = "http://localhost:3000/api/auth";

export const loginUser = async (email, password) => {
  const response = await fetch(`${API_URL}/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ email, password }),
  });
  const data = await response.json();
  if (!response.ok) throw new Error(data.message);
  return data;
};

export const registerUser = async (full_name, email, password, role, phone) => {
  const response = await fetch(`${API_URL}/register`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ full_name, email, password, role, phone }),
  });
  const data = await response.json();
  if (!response.ok) throw new Error(data.message);
  return data;
};
```

Other service modules follow the same `fetch`-based pattern, constructing their URLs relative to `http://localhost:3000/api`.

## JWT authentication

After a successful login or registration, the backend returns a JWT and a user object. The frontend stores both in `localStorage` under fixed keys:

| Key | Value |
|---|---|
| `"token"` | The raw JWT string sent as a `Bearer` token on subsequent requests |
| `"user"` | A JSON-serialised object containing the user's id, name, email, and role |

Components that need the current user read from `localStorage.getItem("user")` and parse the JSON. Protected routes check for the presence of `"token"` to determine whether the user is authenticated.

## Global session interceptor

`App.jsx` installs a monkey-patch over `window.fetch` on mount to handle expired sessions automatically. When any request to `localhost:3000` returns a `401 Unauthorized` response, the interceptor clears both `localStorage` keys and navigates the user to `/auth`:

```javascript src/App.jsx
useEffect(() => {
  const originalFetch = window.fetch;

  window.fetch = async (...args) => {
    const response = await originalFetch(...args);

    if (response.status === 401) {
      const url = args[0];
      if (typeof url === "string" && url.includes("localhost:3000")) {
        localStorage.removeItem("token");
        localStorage.removeItem("user");
        window.location.href = "/auth";
      }
    }

    return response;
  };

  return () => {
    window.fetch = originalFetch;
  };
}, []);
```

The interceptor is scoped to requests that target `localhost:3000`, so third-party fetches (e.g., Google Fonts) are not affected. The original `fetch` is restored when the component unmounts.

## Splash screen

On first load, `App.jsx` renders the `<SplashScreen />` component while the application initialises. The splash screen is hidden after 2.5 seconds via a `setTimeout`:

```javascript src/App.jsx
const [showSplash, setShowSplash] = useState(true);

useEffect(() => {
  const timer = setTimeout(() => setShowSplash(false), 2500);
  return () => clearTimeout(timer);
}, []);
```

The `SplashScreen` component is rendered above all routes and is removed from the DOM once `showSplash` becomes `false`.
