This guide walks you through running the Sena Shopping frontend locally. By the end you will have the app running at `http://localhost:5173` and an account registered on the platform.

<Note>
  The frontend requires the backend REST API to be running at `http://localhost:3000` before you start the dev server. Without it, login, registration, and all product data will be unavailable.
</Note>

## Prerequisites

Before you begin, make sure you have the following installed and running:

- **Node.js 18 or later** — the project uses ES modules (`"type": "module"` in `package.json`) and Vite 7, both of which require Node 18+
- **npm, yarn, or pnpm** — any of the three package managers work
- **Backend server** — the API server must be running at `http://localhost:3000`

## Run the app locally

<Steps>
  <Step title="Clone the repository">
    Clone the frontend repository from GitHub:

    ```bash
    git clone https://github.com/djpalacio/SENAshoppingFrontend.git
    cd SENAshoppingFrontend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages. The project has no optional peer dependencies, so any of the three package managers below will produce a working install.

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>

    Key runtime dependencies installed by this step:

    | Package | Version | Purpose |
    |---|---|---|
    | `react` | ^19.2.0 | UI framework |
    | `react-router-dom` | ^7.13.0 | Client-side routing |
    | `react-hot-toast` | ^2.6.0 | Toast notifications |
    | `jspdf` + `jspdf-autotable` | ^4.2.0 + ^5.0.7 | PDF report generation |
    | `lucide-react` | ^0.564.0 | Icon library |
  </Step>

  <Step title="Start the development server">
    Run the Vite dev server:

    ```bash
    npm run dev
    ```

    Vite will start and print a local URL similar to:

    ```
    VITE v7.2.4  ready in 300 ms

      ➜  Local:   http://localhost:5173/
      ➜  Network: use --host to expose
    ```

    Open `http://localhost:5173` in your browser.
  </Step>

  <Step title="Verify the splash screen and homepage">
    When the app first loads, the `<SplashScreen />` component is displayed for **2.5 seconds** while the application initialises. After it disappears, you should see the Sena Shopping homepage with the hero section and the latest products grid.

    If the products grid is empty or shows an error, confirm the backend server is running at `http://localhost:3000`.
  </Step>

  <Step title="Register an account">
    Navigate to `/auth` to create a new account. The registration form (`registerUser` in `src/services/authService.js`) accepts the following fields:

    | Field | Description |
    |---|---|
    | `full_name` | Your display name on the platform |
    | `email` | Used as your login credential |
    | `password` | Stored securely by the backend |
    | `role` | Choose `usuario` (customer) or `emprendedor` (entrepreneur) |
    | `phone` | Optional contact number |

    After successful registration, the backend returns a JWT and user object. The app stores these in `localStorage` under the keys `"token"` and `"user"` respectively, then redirects you to the homepage as an authenticated user.
  </Step>
</Steps>

## Available scripts

All scripts are defined in `package.json` and run through Vite:

| Script | Command | Description |
|---|---|---|
| `dev` | `npm run dev` | Starts the Vite development server with HMR at `http://localhost:5173` |
| `build` | `npm run build` | Compiles and bundles the app into the `dist/` directory for production |
| `preview` | `npm run preview` | Serves the production build locally so you can verify it before deploying |
| `lint` | `npm run lint` | Runs ESLint across all source files using the project's flat config |

<Tip>
  After running `npm run build`, use `npm run preview` to serve the compiled `dist/` output locally. This is the fastest way to catch issues that only appear in the production build — such as missing environment variables, code-splitting problems, or asset path errors — before you deploy.
</Tip>
